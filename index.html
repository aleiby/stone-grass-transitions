The Problem

Bilinear blending: smooth gradient
ignores texture features

Crack-following: boundary traces
natural seams in the stone

Stone textures have natural cracks between individual stones. If we detect those cracks and route transition boundaries through them, the stone-grass edge looks like grass growing between the stones — not an arbitrary line painted across them.

Tile Grid Setup

Tiles with shared edges

The transition system operates on quad tiles with shared edges. Each tile has 4 corners and a geometric center. Adjacent tiles share edge paths. The stone texture tiles independently of the grid — tile boundaries do not align to texture seams.

This works with any quad grid: regular, irregular, or procedural. Our implementation uses a Stalberg irregular dual grid arranged into hexagonal patches, but the technique is grid-topology-agnostic.

The 16 corner configurations reduce to 6 unique cases via rotation. Each corner carries a surface type (stone vs grass) — a 4-bit mask that drives the transition.

Surface Case: Which Corners Are Stone?

Within each grass tile, corners can be stone or grass. The 4-bit SurfaceCase bitmask determines how many transition edges need crack-following paths:

  • 0 solved: all-grass (0000) or all-stone (1111) — no transitions
  • 2 solved: one corner differs — only the two bordering edges are transition edges
  • 4 solved: edge, saddle, or mixed cases

Classification still references all 4 edge metadata slots, but wavefront solve and traceback only run for transition edges. Non-transition edges use default metadata and are never traced.

// Per-instance data packed into INSTANCE_CUSTOM
float surfaceBits  = INSTANCE_CUSTOM.x; // 4-bit mask
float surfaceRot   = INSTANCE_CUSTOM.y; // canonical rotation
float crackPathRow = INSTANCE_CUSTOM.z; // atlas row

Step 1: Crack Detection via Local Contrast

Stone heightmap

Blurred (local average)

Detected cracks

float h_pixel = textureLod(stone_height, uv, 0.0).r;       // fine detail
float h_blur  = textureLod(stone_height, uv, blur_mip).r;  // local average
float crack_depth = h_blur - h_pixel;                      // positive = in a crack
float in_crack = smoothstep(midpoint - spread/2, midpoint + spread/2, crack_depth);

Step 2: Center Snapping

Drag the shared center vertex to reshape tiles

From geometric to crack center

Each tile's geometric center (centroid of its 4 corners) typically lands on solid stone, since stones are larger than the gaps between them. We search the neighborhood for the best crack pixel.

Two-tier scoring: Any crack pixel beats any stone pixel (cost 0 vs 1000). Among crack pixels, prefer the closest to the geometric center. This ensures edge paths start and end inside the crack network.

float in_crack = smoothstep(mid - s, mid + s,
                          h_blur - h_pixel);
float is_stone = in_crack < 0.5 ? 1.0 : 0.0;
float score = is_stone * 1000.0 + dist;

The search is sparse: step size scales with tile edge length, so the 17×17 grid always covers ~50% of the tile from center. Larger tiles sample fewer texels per step. This is fine because cracks are wide features—sub-texel precision isn't needed to land inside one.

GPU parallelism

One workgroup of 64 threads per tile. Each thread takes one row of the 17-column search grid, finds its row-best score, and writes it to shared memory. Thread 0 picks the global winner, then the winning thread re-derives its best UV and writes the output—17 texture reads per active thread.

Step 3: Edge Pathfinding — Corridor Wavefront

2D shortest path in a corridor

For each shared edge, an oriented texel corridor is built between the two tile centers in texture space. An 8-neighbor weighted relaxation finds the shortest path through the crack cost field — no centerline bias, no lane constraints.

The wavefront can follow cracks that bend, fork, or drift away from the chord. The result is projected onto the edge chord as a t_upper offset table that the classifier samples to determine side-of-path.

// 8-neighbor relaxation per corridor texel
float move_len = n < 4 ? 1.0 : sqrt(2.0);
float edge_cost = move_len * 0.5
  * (self_cost + neighbor_cost);
float cand = dist[neighbor] + edge_cost;
if (cand < best) {
  best = cand;
  best_pred = neighbor;
}

A cheaper fallback uses a fixed 64-lane monotonic DP sweep along the chord, but it cannot represent 2D detours. The narrow corridor keeps the wavefront solve area small regardless of edge length.

Step 4a: Shared Edges & Signed References

Edge paths are shared

Each crack-following path connects two adjacent tile centers. Both tiles reference the same path data, but from opposite perspectives.

The sref sign

A signed edge reference tells each tile how to interpret the shared path:

  • sref > 0: path origin (s=0) is at our center
  • sref < 0: path was created from the neighbor's perspective — flip s_local = 1 - s_param and negate the side test

Without this normalization, the positive/negative side of the path inverts for half the tiles, creating seams at every shared edge.

// Normalize s so s=0 is always at OUR center
s_local = sref < 0 ? (1.0 - s_param) : s_param;
// Flip side test for reversed references
if (sref < 0) raw_side = -raw_side;

Step 4b: The Sequential Overwrite Classifier

All edges

Like closing a box

Each edge claims its territory the way box flaps fold shut — one at a time, each overlapping the last. We process edges S➔E➔N➔W in sequence; each edge overwrites the previous classification for every pixel on its positive side.

The core loop

Start assigned to the S edge. For each subsequent edge, if the pixel is on its positive side and passes the guards, that edge takes ownership.

const int sector_corner[4] = {1,2,3,0};
// S→BR  E→TR  N→TL  W→BL
int assigned = 0;
for (int i = 1; i < 4; i++) {
  if (abs(srefs[i]) == 0) continue;
  if (raw_sides[i] <= 0.0) continue;
  // domain guard (next slide)
  // wrap guard (next slide)
  assigned = i;
}
corner = sector_corner[assigned];

Step 4c: Domain & Wrap Guards

Domain guard

The path starts after the nominal edge endpoint when center snapping lands inside the corridor. The runtime stores a lower s bound; for reversed edge references that lower bound is 1.0 − path_max_s. The upper end stays open so W can fill until it meets S at the wrap seam:

float s_lo = reversed ? 1.0 - path_max_s : path_min_s;
bool new_in_range = s_local[i] > s_lo[i];
bool cur_in_range = s_local[assigned] > s_lo[assigned]
                 && raw_sides[assigned] > 0.0;
if (!new_in_range && cur_in_range)
  continue;

Wrap guard

The loop processes S➔E➔N➔W. Without protection, W (the last edge) could overwrite S (the first) — creating artifacts where two nearly-parallel paths compete:

if (i == 3 && assigned == 0
    && raw_sides[0] > 0.0)
  continue;

Toggle each guard to see the artifacts it prevents. Disable s_range_min to fall back to the infinite half-plane test. Show out-of-range dims pixels before the path starts.

Step 5: Per-Patch Mask Generation

Raw classification

After Gaussian blur

Applied to stone texture

The classify shader writes an R8 texture in stone-texture UV space (not screen space). Each texel finds its owning tile via point-in-quad, runs the sequential overwrite classifier, and writes the corner's stone flag. A separable Gaussian blur softens the transition. The fragment shader samples with a single texture() call.

Performance & Implementation Notes

Compute Budget

  • Center snapping: 1 workgroup per tile, 64 threads. 17×17 candidate grid; each candidate evaluates local height contrast from fine height and blurred height (two mips).
  • Wavefront pathfinding: Corridor atlas packed per-edge, 16×16 workgroups. Fixed-width corridors (tunable pad, currently 64 texels), ~15 iterative relaxation passes.
  • Classification: 8×8 workgroups over mask texture. Point-in-quad test iterates all tiles (use spatial acceleration for large patches).
  • Blur: Separable Gaussian, radius 4. Two passes.
  • Typical patch: ~50 tiles, ~80 edges, 1024×1024 mask. Runs once at load time; suitable for real-time updates when terrain changes

Memory

  • Corridor atlas: packed edge rects, R32F distance + R8UI predecessor (transient, freed after traceback)
  • Edge output: t_upper offsets per chord texel, R16F + edge metadata RGBA32F
  • Crack mask: 1024×1024, R8 + generated mips = ~1.3MB per patch

Integration Checklist

  • Need a stone heightmap with visible cracks (most stone PBR textures work)
  • Need per-tile corner data: 4 world-space positions + stone/grass flags
  • Need adjacency: which tiles share edges (for edge list)
  • Compute pathfinding once at terrain load time (not per-frame)
  • Fragment shader cost: 1 texture sample + smoothstep (negligible)

Corridor Wavefront Solver

Each edge gets a fixed-width oriented corridor (tunable pad, currently 64 texels corridor + 32 texels endpoint). An iterative 8-neighbor relaxation on the corridor atlas finds the shortest crack-following path; traceback runs only for transition edges and projects path points onto the chord as t_upper offsets.

Fixed-width corridors keep the solve area bounded regardless of edge length. Mip generation after blur enables LOD-aware sampling in the fragment shader.
